# Claim Types and Boundary Discipline in Thalean Relational Theory

Status: working discipline note
Date: 2026-06-01
Scope: proof-status boundaries, claim typing, and responsible presentation for TRT

## 1. Purpose

Thalean Relational Theory is intentionally broad.

It includes philosophical framing, finite graph theory, modal logic, computational experiments, public essays, witness artifacts, speculative bridges, and physical analogies.

That breadth is powerful, but it creates a risk:

different kinds of claims can be accidentally blurred.

This note exists to prevent that.

TRT should remain ambitious without becoming careless. The way to do that is to separate claim types, name proof status clearly, and keep every branch tied to its proper witness discipline.

Core rule:

Do not let the umbrella name upgrade the proof status of a claim.

The Thalean label names the scope of the program. It does not make every statement inside the program equally established.

## 2. Why boundary discipline matters

TRT is built around boundary, frame, admissibility, and invariant.

That means TRT must practice what it teaches.

A claim needs a boundary.

A theorem is not a metaphor.
A metaphor is not an experiment.
An experiment is not a proof.
A computational witness is not automatically a theorem.
A philosophical proposal is not automatically a formal system.
A physical analogy is not automatically a physical model.
A public-facing phrase is not automatically a technical definition.

Boundary discipline keeps these forms distinct while allowing them to inform each other.

The goal is not to weaken the theory.

The goal is to make the theory accountable.

## 3. Core distinction

Every TRT statement should be understood as belonging to at least one of the following claim types:

1. Philosophical proposal
2. Formal definition
3. Finite graph theorem
4. Computational verification
5. Computational experiment
6. Visual witness
7. Methodological rule
8. Physical analogy
9. Literature synthesis
10. Speculative bridge
11. Public metaphor
12. Naming convention
13. Working conjecture
14. Locked internal terminology
15. Historical framing

Each type has different standards of support.

## 4. Philosophical proposal

A philosophical proposal introduces a way of thinking, interpreting, or organizing inquiry.

Example:

Existential relativity is the view that identity is not an isolated property of a thing, but a relational invariant revealed across frames.

Support standard:

clarity,
coherence,
usefulness,
relationship to existing traditions,
resistance to obvious confusion,
and ability to guide further formal or empirical work.

Boundary:

A philosophical proposal is not a proof.

It may motivate formal definitions and mathematical models, but it does not establish them by itself.

## 5. Formal definition

A formal definition specifies a term or structure inside a chosen language.

Example:

A relational frame may be specified as S = (W, R, L, A, Adm), where W is a state set, R is a family of typed relations, L is a labeling structure, A is an action set, and Adm is an admissibility predicate.

Support standard:

precision,
non-contradiction,
fitness for intended use,
compatibility with existing definitions,
and ability to support theorems, examples, or computations.

Boundary:

A definition is not a discovery by itself.

It becomes useful when it organizes claims, permits tests, or supports theorem work.

## 6. Finite graph theorem

A finite graph theorem is a mathematical claim about a graph, matrix, invariant, construction, or finite structure that has been proved or verified under explicit assumptions.

Support standard:

clear statement,
clear hypotheses,
proof or exhaustive finite verification,
reproducible artifact if computational,
and separation from interpretive language.

Boundary:

A finite graph theorem does not automatically imply a physical theory, metaphysical doctrine, or public metaphor.

It can motivate those things, but it does not prove them.

## 7. Computational verification

Computational verification is a reproducible check of a finite claim.

Example:

A script verifies that a matrix identity holds for a specified finite graph object.

Support standard:

source code,
input artifact,
output artifact,
hash or reproducibility note where useful,
clear environment assumptions,
and a statement of exactly what was checked.

Boundary:

Computational verification is only as broad as the checked case.

It should not be presented as a general theorem unless the computation exhausts the relevant space or is connected to a proof.

## 8. Computational experiment

A computational experiment explores behavior under simulation, perturbation, search, or statistical comparison.

Support standard:

code,
parameters,
random seed where relevant,
artifact outputs,
controls,
null comparisons,
and cautious interpretation.

Boundary:

A computational experiment can suggest a pattern.

It does not establish a theorem unless turned into proof or exhaustive verification.

It does not establish a physical claim unless connected to empirical data and accepted modeling assumptions.

## 9. Visual witness

A visual witness is a diagram, rendering, graph layout, animation, table, plot, or interface that makes a structure inspectable.

Support standard:

source data,
clear labels,
known projection limits,
and explanation of what the visualization does and does not show.

Boundary:

A visualization is not the object.

A projection is not the full structure.

A compelling image is not a proof.

A visual witness should be treated as a witness surface, not as final evidence by itself.

## 10. Methodological rule

A methodological rule describes how inquiry should proceed.

Example:

Do not protect the idea. Make the idea protect itself.

Support standard:

practical usefulness,
consistency with CoRI principles,
ability to prevent error,
and successful use in workflows.

Boundary:

A methodological rule guides practice. It is not a theorem about reality.

It can become part of institutional method without pretending to be a formal law.

## 11. Physical analogy

A physical analogy compares a TRT structure to a physical process or concept.

Example:

A graph boundary may be compared to a membrane, interface, event horizon, acoustic boundary, or measurement surface.

Support standard:

explicit mapping,
explicit non-mapping,
clear statement of what is being compared,
and clear warning where the analogy stops.

Boundary:

A physical analogy is not a physical model.

It should never be presented as empirical support unless backed by data, equations, and domain-appropriate validation.

## 12. Literature synthesis

A literature synthesis organizes existing sources, claims, equations, measurements, or interpretations.

Support standard:

primary sources,
stable identifiers where possible,
accurate summaries,
explicit evidence class,
separation of source claims from project interpretation,
and conservative language.

Boundary:

A synthesis is not new experimental evidence.

It can support a model comparison or research question, but it should not overstate what the sources establish.

## 13. Speculative bridge

A speculative bridge proposes that a TRT concept may illuminate another domain.

Example:

A finite relational boundary concept may offer a useful interpretive lens for a physical boundary-flux problem.

Support standard:

caution,
explicit forbidden mappings,
clear proof-status label,
and separation from theorem-backed or empirically established claims.

Boundary:

A speculative bridge is not established theory.

It should be presented as a research direction, not as a conclusion.

## 14. Public metaphor

A public metaphor helps readers understand an idea.

Example:

Boundary is where relation becomes regulated exchange.

Support standard:

clarity,
memorability,
non-misleading fit,
and ability to point back to stricter definitions.

Boundary:

A metaphor is not a definition.

It can introduce, but it must not replace, technical precision.

## 15. Naming convention

A naming convention assigns a stable label to a concept, artifact, or object.

Example:

Thalean Relational Theory names the umbrella program.
Thalean Graph Theory names the graph-theoretic subprogram.
Existential relativity names the philosophical thesis.
Discrete Action Theory names the operational dynamics.

Support standard:

consistency,
scope clarity,
low ambiguity,
and usefulness across notes.

Boundary:

A name is not a proof.

A name should organize inquiry, not inflate certainty.

## 16. Working conjecture

A working conjecture is a claim believed to be promising but not yet proved.

Support standard:

explicit conjecture label,
motivation,
known evidence,
known counterchecks,
open tests,
and failure conditions.

Boundary:

A conjecture should not be written as if established.

Its value is in guiding tests.

## 17. Locked internal terminology

Locked internal terminology is a vocabulary decision for project coherence.

Example:

A thalion refers to the G60 form, the concrete 60-state transport organism or instance with its full relationships and properties.

Support standard:

explicit memory or repo note,
consistent use,
and explanation where public readers may be confused.

Boundary:

Locked terminology stabilizes project language. It does not imply external acceptance.

When writing for public audiences, define locked terms before relying on them.

## 18. Historical framing

Historical framing places the project in a lineage or story.

Example:

The Thalean label points to Thales as a historical symbol for the movement from inherited image toward accountable structure.

Support standard:

accuracy,
modesty,
and avoidance of overclaiming direct continuity.

Boundary:

Historical framing provides orientation. It does not prove the theory.

## 19. Claim label system

TRT notes should use explicit claim labels when helpful.

Suggested labels:

[PHILOSOPHICAL PROPOSAL]
[FORMAL DEFINITION]
[GRAPH THEOREM]
[COMPUTATIONAL VERIFICATION]
[COMPUTATIONAL EXPERIMENT]
[VISUAL WITNESS]
[METHOD RULE]
[PHYSICAL ANALOGY]
[LITERATURE SYNTHESIS]
[SPECULATIVE BRIDGE]
[PUBLIC METAPHOR]
[NAMING CONVENTION]
[WORKING CONJECTURE]
[LOCKED TERMINOLOGY]
[HISTORICAL FRAMING]

A single passage may carry more than one label.

When in doubt, use the weaker label.

## 20. The ladder of strength

TRT should treat claim strength as a ladder.

A possible ladder:

1. Phrase or metaphor
2. Naming convention
3. Philosophical proposal
4. Working definition
5. Formal definition
6. Worked example
7. Computational experiment
8. Computational verification
9. Finite theorem
10. General theorem
11. Empirical model
12. Empirically validated model

Do not skip ladder rungs in public presentation.

A metaphor may motivate a definition.
A definition may support a worked example.
A worked example may motivate a conjecture.
A conjecture may guide computation.
Computation may suggest a theorem.
A theorem may support a method.
A method may guide empirical work.

But each transition must be earned.

## 21. Boundary discipline for TRT and TGT

TRT is the umbrella.

TGT is the mathematical graph core.

The relationship should be stated carefully.

Allowed:

TGT provides finite graph witnesses for TRT ideas.

Allowed:

TRT supplies philosophical and methodological framing for TGT.

Not allowed:

Because a TGT graph theorem holds, a TRT metaphysical thesis is proved.

Not allowed:

Because a TRT philosophical thesis is compelling, a TGT mathematical claim is established.

The bridge is productive, but the proof statuses are distinct.

## 22. Boundary discipline for physics bridges

TRT may inspire physical analogies and research questions.

But physical claims require physical discipline.

A physical bridge should specify:

the TRT concept,
the physical concept,
the proposed mapping,
the forbidden mappings,
the evidence source,
the model limits,
and the proof status.

Physical domains need their own standards:

measurement,
equation,
units,
prediction,
uncertainty,
replication,
literature support,
and empirical constraint.

The phrase "this resembles" must not become "this explains."

## 23. Boundary discipline for AI and coherence work

TRT connects naturally to AI coherence, local mimicry, and surface testing.

But AI claims also need type discipline.

A coherence-gap theorem in finite graphs is not automatically a theorem about language models.

It may provide:

a model,
a metaphor,
a test method,
a mathematical analogy,
or a design principle.

To become an AI claim, it needs AI-specific experiments and evidence.

Correct phrasing:

This finite graph result gives a formal toy model for local mimicry and global coherence failure.

Overclaiming phrasing:

This proves how AI systems behave.

## 24. Boundary discipline for public writing

Public writing may use memorable phrases.

But public phrases should point back to technical boundaries.

Good public phrase:

Objectivity is not the absence of perspective. It is the survival of structure across perspectives.

Technical backing:

Objectivity is treated as frame-crossing invariance under a disciplined family of relational transformations.

Good public phrase:

A boundary is an admissibility surface.

Technical backing:

A boundary regulates which exchanges, actions, or transformations between inside and outside preserve the body or frame.

Public writing can be poetic, but it should not erase claim type.

## 25. Minimum claim packet

For serious TRT claims, use a minimum claim packet.

A claim packet should include:

Claim:
What is being asserted?

Type:
What kind of claim is it?

Frame:
In what relational frame does it apply?

Boundary:
What is included and excluded?

Admissible transformations:
Which changes are allowed?

Invariant:
What is claimed to survive?

Witness:
What artifact, proof, computation, source, or argument supports it?

Limit:
What does the claim not establish?

This packet should become a standard discipline.

## 26. Example claim packets

Example 1:

Claim:
A subjective report is a frame-local witness.

Type:
Philosophical proposal / formal framing.

Frame:
TRT relational-frame language.

Boundary:
Applies to epistemic interpretation, not clinical psychology by itself.

Admissible transformations:
Changed observer, changed boundary, changed interpretive frame.

Invariant:
The report remains a trace from inside a boundary.

Witness:
Conceptual analysis in notes/005.

Limit:
Does not establish the truth of any particular subjective report.

Example 2:

Claim:
A thalion refers to the G60 form, not the G15 quotient.

Type:
Locked terminology.

Frame:
TGT project vocabulary.

Boundary:
Internal project terminology unless defined for public readers.

Admissible transformations:
Quotienting, projection, compression.

Invariant:
The thalion is the full concrete 60-state organism.

Witness:
Glossary note.

Limit:
Does not claim external naming adoption.

Example 3:

Claim:
A coherence gap is a mismatch between local appearance and global structure.

Type:
Formal/methodological framing.

Frame:
CoRI and finite graph coherence testing.

Boundary:
Applies to modeled surfaces and tested relational structures.

Admissible transformations:
Changed viewpoint, perturbation, relabeling, wider-frame checks.

Invariant:
Accountable structure should survive relevant transformations.

Witness:
Coherence-gap paper and TRT notes.

Limit:
Does not automatically characterize every AI or philosophical failure.

## 27. Failure modes this note prevents

Failure mode 1:
Metaphor inflation.

A useful phrase becomes treated as proof.

Failure mode 2:
Theorem leakage.

A finite graph theorem is treated as proving a metaphysical or physical claim.

Failure mode 3:
Speculation hardening.

A speculative bridge becomes repeated as settled fact.

Failure mode 4:
Visual seduction.

A compelling visualization is treated as the object itself.

Failure mode 5:
Vocabulary capture.

A coined term creates an illusion of explanation.

Failure mode 6:
Domain trespass.

A method from one domain is applied to another without rebuilding evidence.

Failure mode 7:
Boundary erasure.

The reader cannot tell what is inside the claim and what is outside it.

TRT should be designed to resist these failures.

## 28. Positive version

Boundary discipline is not only defensive.

It is constructive.

Clear claim types let the whole body of work grow without confusion.

Philosophy can motivate formalism.
Formalism can guide computation.
Computation can produce witnesses.
Witnesses can support papers.
Papers can refine public language.
Public language can attract critique.
Critique can sharpen definitions.
Definitions can produce theorems.

The work can be broad because the boundaries are explicit.

## 29. Summary

Thalean Relational Theory requires claim discipline because it intentionally spans philosophy, mathematics, computation, public writing, and speculative bridgework.

The rule is simple:

Name the claim type.
Declare the boundary.
Identify the witness.
State the limit.

This protects the project from overclaiming while preserving its ambition.
